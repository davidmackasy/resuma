SECTION 1: INTELLIGENT JOB-BASED RELEVANCE FILTERING

PROBLEM
Currently resume uses profile content but does not properly filter irrelevant experience.

REQUIREMENT
When user pastes job description:

System must analyze job requirements

Score each experience role and bullet

Include only relevant roles/bullets

Reorder by relevance

Exclude irrelevant experience from final output

DO NOT modify applykit_profiles

IMPLEMENTATION LOGIC

Step 1: Extract Job Signals

From job_description generate structured JSON:

{
job_category,
required_skills[],
preferred_skills[],
responsibilities[],
keywords[],
seniority_level,
industry_type
}

Example:
If construction labor job:

keywords: construction, manual labor, heavy lifting, tools

industry_type: labor

required_skills: physical stamina, teamwork, safety compliance

Step 2: Score Profile Experience

For each role in applykit_profiles.experience:

Calculate relevance score:

Keyword overlap count

Skill match count

Industry similarity

Responsibility similarity

Score formula example:
score = (keyword_matches * 2) + (skill_matches * 3) + (industry_match ? 5 : 0)

Sort roles by score descending.

Step 3: Filter Logic

Rules:

Include top 2–4 most relevant roles

Exclude roles with score below threshold

If user applying to labor job:
Exclude unrelated doctor/office roles unless they share relevant skills

If tech job:
Exclude unrelated manual labor roles

Do NOT delete them from profile.
Only exclude from generated output.

Step 4: Bullet Selection

For each included role:

Score each bullet

Keep top 3–5 relevant bullets

Rewrite bullets for clarity, not fabrication

Remove fluff

Keep measurable results only if present
