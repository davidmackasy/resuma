SECTION 3: COMMON FAILURE CAUSE

You are probably doing:

insert into applykit_documents

Instead of updating existing ones.

If doc_type = resume already exists,
unique constraint or conflict may occur.

Fix:

Use update, not insert.

Example:

await supabase
.from("applykit_documents")
.update({
content_json: newResumeJson,
content_md: newResumeMd,
updated_at: new Date()
})
.eq("application_id", applicationId)
.eq("doc_type", "resume")

Repeat for:
cover_letter
email

Do NOT insert new row.
