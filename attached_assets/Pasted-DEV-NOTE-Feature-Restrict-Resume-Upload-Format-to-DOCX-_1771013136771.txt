DEV NOTE
Feature: Restrict Resume Upload Format to DOCX Only
Scope: Resume Upload Flow Only
Do Not Affect: Resume generation, resume download (PDF/DOC), DOCX parser logic, auth, Stripe, admin, limits, storage, or any other document features.

Objective
Remove PDF upload support when users upload their resume.
Allow only .docx files.
If user attempts to upload a PDF, show a professional UI-matching notification.
Do not break or modify existing DOCX functionality.

FRONTEND CHANGES

A. File Input Restriction

Locate resume upload component. Likely something like:

<input type="file" />

Update accept attribute to:

accept=".docx,application/vnd.openxmlformats-officedocument.wordprocessingml.document"

Remove:

application/pdf

.pdf

This ensures OS file picker filters PDFs out visually.

B. Client-Side Validation

Even with accept filter, user may bypass via drag and drop.

Add strict validation before upload request:

Pseudo logic:

if (file.type !== "application/vnd.openxmlformats-officedocument.wordprocessingml.document") {
showFormatError();
return;
}

Do not check only extension. Check MIME type first. Fallback to extension if needed.

Allowed:
application/vnd.openxmlformats-officedocument.wordprocessingml.document

Rejected:
application/pdf
Anything else

C. Professional Notification UI

Do NOT use red error background.
Do NOT use destructive alert styling.
Do NOT break existing UI theme.

Instead:

Use your app’s neutral info notification style:

Same border radius as cards

Same background as modal surface

Subtle left accent border in primary brand color

Normal text color

Small icon if already used elsewhere

Message copy:

Title:
Unsupported File Format

Body:

Use neutral toast variant.

Make sure:

Works on mobile

No layout shift

Locate resume upload endpoint.

if (file.mimetype !== "application/vnd.openxmlformats-officedocument.wordprocessingml.document") {
