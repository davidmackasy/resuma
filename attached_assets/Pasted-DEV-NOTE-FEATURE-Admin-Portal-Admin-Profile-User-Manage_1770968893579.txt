DATABASE CHANGES (NEW TABLES ONLY)

A) applykit_admins
Purpose: admin allowlist + role

Columns

id uuid pk default gen_random_uuid()

user_id uuid unique not null references auth.users(id) on delete cascade

email text not null

role text not null default 'admin' // 'owner' | 'admin' | 'support'

is_active boolean not null default true

created_at timestamptz default now()

created_by uuid null references auth.users(id)

Notes

Store email for display and quick lookup

user_id is the true key

B) applykit_user_flags
Purpose: moderation and account actions

Columns

user_id uuid pk references auth.users(id) on delete cascade

is_banned boolean not null default false

ban_reason text null

banned_at timestamptz null

banned_by uuid null references auth.users(id)

notes text null

updated_at timestamptz default now()

C) applykit_user_overrides
Purpose: give extra limits, override plan, admin unlimited

Columns

user_id uuid pk references auth.users(id) on delete cascade

extra_applications int not null default 0

extra_regenerations int not null default 0

force_unlimited boolean not null default false

override_expires_at timestamptz null

updated_at timestamptz default now()

updated_by uuid null references auth.users(id)

D) applykit_events (optional but strongly recommended)
Purpose: analytics without relying on logs

Columns

id uuid pk default gen_random_uuid()

user_id uuid null references auth.users(id)

event_type text not null // 'signup' | 'login' | 'analysis' | 'generate' | 'regenerate' | 'download_pdf' | 'subscribe' | 'cancel'

metadata jsonb default '{}'

created_at timestamptz default now()

ADMIN UNLIMITED ACCESS

Where plan limits are enforced today (usage checking function), update logic:

Return unlimited = true

Else check applykit_user_overrides.force_unlimited true and not expired
Return unlimited = true

Else apply normal Stripe plan limits + extra allowances:
