DEV NOTE
PROJECT: Premium Resume Tailor Web App
GOAL: User uploads master resume once. User pastes job description. App generates 3 outputs: tailored resume (ATS-safe), cover letter, follow-up email. User can preview and download PDFs. App includes dashboard, profile, history, settings, and Stripe plans.
CRITICAL: Do not break authentication. Do not break Stripe. Do not change unrelated pages/modules. Build this as a standalone app module with isolated routes, isolated tables, isolated APIs.

CORE USER FLOW
A) Onboarding

User signs up

Redirect to /applykit/app/profile

Prompt: Upload master resume (PDF/DOCX)

Parse + store structured profile

User can edit extracted content before generating anything

B) New Application

User goes to /applykit/app/new

Paste job description

Optional: company name, role title, location, hiring manager name, link

Select format template (4–6 locked templates)

Select tone (Professional, Confident, Warm, Direct)

Click Generate

System creates Application record + three Documents (resume, cover letter, email)

User lands on Application detail page with previews and downloads

C) History

Dashboard shows recent applications

History list shows all applications with status tags

Re-open any application, regenerate, or download again

FILE UPLOAD AND PARSING
Upload

Accept PDF and DOCX

Store original file in Supabase Storage: applykit/master_resumes/{user_id}/v{resume_version}.{ext}

Save path in applykit_profiles.resume_file_url

Parsing strategy (MVP)

If DOCX: use docx parser (server side) to extract text reliably

If PDF: extract text with a PDF text extractor (server side)

Convert to a normalized “profile draft”:

Contact block

Experience roles and bullets

Skills list

Education

Show extracted data in UI for user to fix before first generation
Important: parsing will never be perfect. The UI editor is mandatory.

GENERATION ARCHITECTURE (PROMPTING)
Do not “rewrite everything blindly”. Use a controlled pipeline.

Step 1: Job extraction
Input: job_description
Output JSON:

role_title

seniority

required_skills[]

preferred_skills[]

responsibilities[]

keywords[]

tone_hint

Step 2: Profile selection
Input: applykit_profiles (structured) + job JSON
Output JSON:

selected_experience_roles[]

selected_bullets_by_role

selected_skills_grouped

suggested_summary_points

gaps_or_unknowns (do not invent, just flag)

Step 3: Generate each doc (resume, cover, email)
Use the selected JSON so the model writes only from approved facts.
Hard rule: no fabricated companies, dates, degrees, awards.

Resume output requirements

1 page preferred, 2 pages allowed for senior roles

3–5 bullets per role max

Active verbs, measurable outcomes if present in source bullets

No buzzword soup

ATS-safe formatting

Cover letter requirements

max 250–350 words unless user chooses “long”

tailored opening referencing company/role

1–2 proof points aligned to responsibilities

clear close

Email requirements

initial application email: 60–120 words

follow-up email: 60–120 words

subject line included

professional and human

Store output in:

content_json (structured sections)

content_md (human readable)

PDF RENDERING (CRITICAL FOR “PREMIUM”)
Requirement: “download in PDF format” with consistent design.

Recommended approach (stable)
Option A: Server-side HTML to PDF via Playwright/Puppeteer

Build HTML templates per template_id

Render to PDF on server

Upload to storage and store pdf_url
Pros: best typography control, consistent output

Option B: @react-pdf/renderer

Create PDF components per template
Pros: pure React, simpler deploy
Cons: fonts/layout can be tricky

Pick one and standardize. Do not mix.

Also provide:

Copy text (for email)

Optional DOCX export later (phase 2)

EDGE CASES TO HANDLE

User has no profile yet: redirect to profile upload

Very long job description: truncate safely after N chars and keep requirements section

Missing company/role: extract from job description, else leave blank cleanly

PDF rendering failure: still allow copy text while PDF retries

Multiple resume versions: increment resume_version and keep prior applications intact
